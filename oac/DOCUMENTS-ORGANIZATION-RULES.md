# Правила организации документов по ОАЦ (Оперативно-аналитический центр)

**Версия:** 1.0

## 📋 СТРУКТУРА ХРАНЕНИЯ ДОКУМЕНТОВ

### 1. **origin-docs/** - Оригинальные нормативные документы

**Назначение:** Хранение оригинальных текстов законов, указов и приказов в формате Markdown или PDF.

**Содержимое:**
```
origin-docs/
├── README.md                          # Описание папки и навигация
├── NORMATIVNYE-DOKUMENTY-REGISTER.md  # Полный реестр с анализом
├── 449.md                             # Указ Президента № 449 (текст)
├── 195.md                             # Приказ ОАЦ № 195 (текст)
├── 99.md                              # Закон № 99-З (текст, если есть)
├── zakon 99-3.pdf                     # Закон № 99-З (PDF оригинал)
├── Chek_list_po_provedeniju_proverok_operatorov.docx  # Чек-лист проверок
└── PRIKAZ-195-ANALYSIS-TEMPLATE.md    # Шаблон для анализа новых документов
```

**Правила:**
- ✅ Только **оригинальные тексты** нормативных актов
- ✅ Формат: **.md** (предпочтительно) или **.pdf** (как резервная копия)
- ❌ НЕ размещать аналитику, интерпретации или проектные документы
- ✅ Каждый документ должен иметь четкое название с номером и датой

---

### 2. **oac/docs/** - Документы compliance проекта

**Назначение:** Практические документы, разработанные для соответствия требованиям ОАЦ.

**Структура нумерации:**
```
oac/docs/
├── 01-act-class-3in.md                # Акт отнесения к классу защиты
├── 02-structural-schema.md            # Структурная схема ИС
├── 03-logical-schema.md               # Логическая схема ИС
├── 04-privacy-policy.md               # Политика обработки ПД
├── 05-infosec-policy.md               # Политика информационной безопасности
├── 06-tech-spec.md                    # Техническое задание на СЗИ
├── 07-ib-monitoring-reglament.md      # Регламент мониторинга ИБ
├── 08-backup-reglament.md             # Регламент резервного копирования
├── 09-data-retention-reglament.md     # Регламент хранения данных
├── 10-encryption-policy.md            # Политика шифрования
├── 11-antivirus-reglament.md          # Регламент антивирусной защиты
├── 12-ids-ips-reglament.md            # Регламент IDS/IPS
└── 13-vuln-scan-reglament.md          # Регламент сканирования уязвимостей
```

**Правила:**
- ✅ Строгая **нумерация** (01-13) для поддержания порядка
- ✅ Каждый документ должен ссылаться на соответствующие пункты нормативных актов
- ✅ Использовать шаблоны с заполнителями `[НАИМЕНОВАНИЕ]`, `[ДАТА]` и т.д.
- ✅ Обновлять при изменении требований или архитектуры системы
- ❌ НЕ дублировать текст законов - только ссылки на них

### 3. **oac/audits/** - Отчеты об аудитах и проверках

**Назначение:** Результаты внутренних и внешних проверок, тестирований.

**Структура:**
```
oac/audits/
├── YYYY-MM-DD_audit-type_description.md  # Отчеты об аудитах
├── vulnerability-scans/                   # Результаты сканирования уязвимостей
├── penetration-tests/                     # Отчеты о тестировании на проникновение
└── compliance-checks/                     # Проверки соответствия требованиям
```

**Правила:**
- ✅ Именование файлов: `YYYY-MM-DD_тип-аудита_описание.md`
- ✅ Включать дату проведения, методологию, результаты, рекомендации
- ✅ Прикреплять технические отчеты (сканы, логи) как приложения
- ✅ Отмечать статус выполнения рекомендаций

## 🎯 ПРАВИЛА СОЗДАНИЯ НОВЫХ ДОКУМЕНТОВ

### При добавлении нового нормативного документа:

1. **Сохранить оригинал** в `origin-docs/`:
   ```bash
   # Если текстовая версия
   cp new-law.md origin-docs/new-law.md
   
   # Если PDF (только как резервная копия)
   cp new-law.pdf origin-docs/new-law.pdf
   ```

2. **Обновить реестр** `origin-docs/NORMATIVNYE-DOKUMENTY-REGISTER.md`:
   - Добавить запись в таблицу нормативных документов
   - Указать статус наличия
   - Добавить ссылки на связанные документы проекта

3. **Создать аналитику** в соответствующей папке `oac/`:
   - Если влияет на compliance → обновить документы в `oac/docs/`
   - Если требует новых процедур → создать в `oac/guides/`
   - Если нужны изменения → добавить в `oac/planning/`

4. **Обновить CHANGELOG** `oac/CHANGELOG.md`:
   - Задокументировать все изменения
   - Указать причину (ссылка на нормативный акт)
   - Отметить затронутые документы

## 🔍 КОНТРОЛЬ КАЧЕСТВА

### Перед коммитом проверить:

1. ✅ Документ находится в правильной папке
2. ✅ Название файла соответствует правилам именования
3. ✅ Есть ссылки на нормативные акты (если применимо)
4. ✅ Обновлен CHANGELOG.md
5. ✅ Нет дублирования информации из других документов
6. ✅ Все placeholder'ы заполнены или отмечены для заполнения
7. ✅ Проверена орфография и форматирование

---

## ⚠️ ЗАПРЕЩЕННЫЕ ДЕЙСТВИЯ

❌ **НЕ хранить** оригиналы законов в `oac/docs/`  
❌ **НЕ смешивать** аналитику с оригинальными текстами  
❌ **НЕ удалять** старые версии без архивации  
❌ **НЕ использовать** неструктурированные названия файлов  
❌ **НЕ забывать** обновлять реестр и CHANGELOG

## 💡 РЕКОМЕНДАЦИИ

### Для эффективной работы:

1. **Регулярная ревизия:** Раз в квартал проверять актуальность всех документов
2. **Версионирование:** Использовать Git для отслеживания изменений
3. **Ссылки:** Всегда указывать источники требований (пункты приказов)
4. **Поиск:** Использовать `grep` для быстрого нахождения требований в oac.md
5. **Шаблоны:** Создавать шаблоны для типовых документов
6. **Автоматизация:** Использовать скрипт `check_normative_docs.py` для проверки полноты

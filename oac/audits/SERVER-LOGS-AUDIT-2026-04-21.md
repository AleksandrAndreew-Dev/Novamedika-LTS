## 🚨 КРИТИЧЕСКИЕ ПРОБЛЕМЫ

### 1. ❌ Ошибка миграции базы данных (КРИТИЧНО)

**Ошибка:**
```
asyncpg.exceptions.UndefinedFunctionError: function pgp_sym_encrypt(text, text) does not exist
```

**Где:** Alembic migration `3b81fefeff37_add_encrypted_fields_for_personal_data.py`

**Причина:**
- PostgreSQL не имеет установленного расширения `pgcrypto`
- Функция `pgp_sym_encrypt()` является частью этого расширения
- Миграция пытается зашифровать существующие данные, но функция недоступна

**Влияние:**
- ⚠️ Backend запустился, но миграция НЕ выполнена (`⚠️ Alembic upgrade failed, continuing without migrations...`)
- Поля для шифрования персональных данных НЕ созданы
- **Система работает без шифрования ПД** - нарушение требований ОАЦ класса 3-ин!

**Решение:**
1. Установить расширение pgcrypto в PostgreSQL:
   ```sql
   CREATE EXTENSION IF NOT EXISTS pgcrypto;
   ```
2. Повторно запустить миграцию:
   ```bash
   docker-compose -f docker-compose.traefik.prod.yml exec backend-prod alembic upgrade head
   ```

### 2. ⚠️ Предупреждения безопасности Celery Worker

**Предупреждения:**
```
SecurityWarning: An entry for the specified gid or egid was not found.
SecurityWarning: You're running the worker with superuser privileges
```

**Причина:**
- Celery worker запущен от root пользователя
- Не найден entry в /etc/group для указанного GID

**Влияние:**
- Потенциальная уязвимость безопасности
- Нарушение принципа минимальных привилегий

**Решение:**
- Запускать Celery worker от непривилегированного пользователя
- Настроить правильные UID/GID в Dockerfile

**Предупреждение:**
```

```

На хост-машине выполнить:
```bash
sudo sysctl vm.overcommit_memory=1
echo 'vm.overcommit_memory=1' | sudo tee -a /etc/sysctl.conf
```

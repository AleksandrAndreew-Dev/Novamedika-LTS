# Анализ требований ОАЦ к внешним ссылкам и переходам на сторонние ресурсы

## 📋 Резюме

**Текущая ситуация:** В приложении есть кнопка "Посмотреть на карте", которая открывает внешний ресурс (Google Maps/Yandex Maps) в новой вкладке браузера.

**Вопрос:** Какие требования ОАЦ применяются к таким внешним переходам?

---

## 🔍 Требования законодательства

### 1. Закон №99-З «О защите персональных данных»

#### Статья 17: Защита персональных данных
> Оператор обязан принимать необходимые правовые, организационные и технические меры для защиты персональных данных от несанкционированного или случайного доступа к ним, уничтожения, изменения, блокирования, копирования, предоставления, распространения персональных данных.

**Применительно к внешним ссылкам:**
- При переходе на внешний ресурс **не должны передаваться персональные данные** пользователей без их согласия
- Внешний сервис не должен получать доступ к данным из вашего приложения

- Ваш сервер **не передаёт данные** на внешние ресурсы
- Это **НЕ является сетевым подключением** вашей инфраструктуры

### Анализ безопасности:

| Параметр | Статус | Комментарий |
|----------|--------|-------------|
| `target="_blank"` | ✅ Правильно | Открывает в новой вкладке |
| `rel="noopener"` | ✅ Правильно | Предотвращает доступ к window.opener |
| `rel="noreferrer"` | ✅ Правильно | Не передаёт Referer header |
| Передача ПД | ✅ Безопасно | Только адрес аптеки (публичные данные) |
| Серверные запросы | ✅ Нет | Переход полностью на клиенте |

## 📊 Оценка рисков

### Риск утечки ПД через внешние ссылки: **НИЗКИЙ** ✅

**Причины:**
1. Передаётся только адрес аптеки (публичная информация)
2. Нет передачи персональных данных пользователей
3. Используется `rel="noopener noreferrer"` (безопасный переход)
4. Переход происходит на стороне клиента (не через ваш сервер)
5. Пользователь явно инициирует переход (клик по кнопке)

### Потенциальные риски:

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Внешний сервис собирает метаданные (IP, User-Agent) | Высокая | Низкая | Нормальная практика, пользователь соглашается с политикой карт |
| Фишинг через подмену ссылки | Низкая | Средняя | HTTPS + проверка URL перед открытием |
| Утечка данных через Referer | Нулевая | - | Предотвращено `rel="noreferrer"` |
| Доступ к window.opener | Нулевая | - | Предотвращено `rel="noopener"` |

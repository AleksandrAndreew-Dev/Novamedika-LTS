**Вывод:** ✅ Все 5 целей имеют документированные правовые основания, соответствующие Закону №99-З. Требование ст. 5, 6 выполнено.

### 1.2. Сроки хранения (статья 4 п. 8 Закона №99-З)

| Категория ПД | Срок | Критерий отсчёта | Статус |
|-------------|------|------------------|--------|
| Q&A пользователей | 1 год | Дата последнего обращения | ✅ |
| Фармацевты | 1 год | Прекращение активности | ✅ |
| Заказы | 3 года | Дата оформления | ✅ |
| Логи ИБ | 1 год | Дата создания (требование ОАЦ) | ✅ |
| CSV-данные аптек | +30 дней после синхронизации | Дата синхронизации | ✅ |
| Refresh tokens | 7 дней | Дата создания | ✅ |
| Audit logs | ≥ 1 года | Дата создания | ✅ |

**Автоматизация:** Celery-задача `cleanup_expired_personal_data` запускается ежедневно в 03:00 (UTC+3) — задокументировано в `09-data-retention-reglament.md`.

**Вывод:** ✅ Сроки установлены для каждой категории, автоматизированы, задокументированы. Соответствует ст. 4 п. 8.

### 1.3. Выявленные пробелы

- ⚠️ **Необходимо документальное подтверждение** включения ИС в Реестр операторов ПД (https://cpd.by) — ст. 17 п. 3 Закона
- ⚠️ **Правовое основание для обработки биометрических данных** не определено (если планируется видеоидентификация)

## 5. Проверка наличия избыточной обработки ПД (статья 4 п. 5 Закона)

### 5.1. Анализ соразмерности и неизбыточности

| Цель обработки | Собираемые ПД | Избыточно? | Обоснование |
|----------------|---------------|------------|-------------|
| **Регистрация фармацевтов** | Telegram ID, ФИО, телефон, статус, квалификация | ⚠️ Частично | Квалификация собирается, но не используется для подбора |
| **Q&A консультации** | Telegram ID, ФИО, телефон, текст вопроса, история | ✅ Нет | Минимально необходимые |
| **Оформление заказов** | Имя, телефон, продукт, аптека, количество | ✅ Нет | Только данные, нужные для брони |
| **Синхронизация с аптеками** | Данные аптек + товары из CSV | ✅ Нет | Данные контрагентов и публичные данные |
| **Обеспечение ИБ** | IP, логи аутентификации, JWT, события | ✅ Нет | Минимально для ИБ |

### 5.2. Выявленная избыточность

**Сомнительная:** Квалификация фармацевта в `qa_pharmacists` собирается при регистрации, но не используется для маршрутизации вопросов. **Рекомендация:** реализовать использование или удалить из формы регистрации.

**Не является избыточным:** Telegram ID, ФИО (для отображения), телефон (для связи), логи ИБ (требование ОАЦ).

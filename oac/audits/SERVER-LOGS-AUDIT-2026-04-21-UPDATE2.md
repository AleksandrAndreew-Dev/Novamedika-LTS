### Проблема роста памяти PostgreSQL

**Аномалия:**
- За 50 минут память выросла с 174 MiB до 327 MiB (+88%)
- При этом размер БД остался 437 MB (без изменений)
- Нет новых записей в таблицах

**Возможные причины:**

1. **Кеширование PostgreSQL:**
   - shared_buffers заполняется
   - OS page cache увеличивается
   - Это нормальное поведение для PostgreSQL

2. **Длительные транзакции:**
   - Незавершенные транзакции держат snapshot
   - Prevent vacuum cleanup
   - Проверить: `SELECT * FROM pg_stat_activity WHERE state != 'idle';`

3. **Autovacuum процессы:**
   - Запущен autovacuum для обслуживания таблиц
   - Временно увеличивает использование памяти

4. **Утечка памяти (маловероятно):**
   - Bug в PostgreSQL или расширении
   - Требует мониторинга

**Рекомендации:**

```sql
-- Проверить активные соединения
SELECT count(*), state FROM pg_stat_activity GROUP BY state;

-- Проверить длительные транзакции
SELECT pid, now() - pg_stat_activity.xact_start AS duration, query
FROM pg_stat_activity
WHERE (now() - pg_stat_activity.xact_start) > interval '5 minutes';

-- Проверить размер shared_buffers
SHOW shared_buffers;

-- Проверить текущее использование памяти
SELECT pg_size_pretty(pg_database_size('novamedika'));
```

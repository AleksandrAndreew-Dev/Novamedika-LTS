# Вопросы для юриста по проекту NovaMedika2

**Дата подготовки:** 4 мая 2026 г.
**Проект:** NovaMedika2 — сервис поиска лекарств и онлайн-консультаций с фармацевтами
**Цель:** Уточнить правовые аспекты для завершения compliance с требованиями ОАЦ (Оператор Активных Компонентов) Республики Беларусь

---

## 📋 Краткое описание проекта

NovaMedika2 — это информационный сервис, который помогает людям:
1. **Находить лекарства** в аптеках города (по названию, производителю, цене)
2. **Получать консультации** от фармацевтов через Telegram (задать вопрос о лекарстве → получить ответ специалиста)
3. **Бронировать лекарства** в конкретных аптеках (оформить заказ → забрать в аптеке)

Сервис работает через:
- Веб-сайт: spravka.novamedika.com
- Telegram бот: @Novamedika_bot
- API для аптек-партнеров (загрузка данных о наличии лекарств)

### 4. ПОЛУЧЕНИЕ СОГЛАСИЯ ПОЛЬЗОВАТЕЛЕЙ

**Вопрос:** Как правильно получать согласие пользователей на обработку их персональных данных?

**Контекст из нашего проекта:**
- Сейчас пользователи начинают пользоваться сервисом без явного согласия:
  - Заходят на сайт spravka.novamedika.com
  - Пишут в Telegram бот @Novamedika_bot команду /start
  - Сразу могут задать вопрос фармацевту
  - Могут оформить заказ на лекарство
- Собираем данные автоматически:
  - Telegram ID (из профиля пользователя)
  - Имя и фамилия (из профиля Telegram)
  - Номер телефона (если пользователь предоставил)
  - Текст вопроса о лекарствах/здоровье
  - История диалогов с фармацевтом
  - Данные заказов (какое лекарство, в какой аптеке, количество)

**Текущая реализация:**
- Политика конфиденциальности размещена на сайте (раздел footer)
- Нет всплывающего окна с согласием при первом использовании
- Нет галочки "Согласен на обработку ПД" при регистрации
- Telegram бот не показывает текст согласия перед началом работы

**Нормативная база:** Статья 5 Закона №99-З требует, чтобы согласие было "свободным, однозначным, информированным".

**Что нужно от юриста:**
- ✅ Требуется ли явное согласие (галочка/кнопка "Согласен") или достаточно implicit consent (использование сервиса = согласие)?
- ✅ Где должно размещаться согласие:
  - При первом входе на сайт (popup)?
  - При команде /start в Telegram боте?
  - Перед отправкой первого вопроса фармацевту?
  - При оформлении заказа на лекарство?
- ✅ Какой текст согласия использовать (шаблон)?
- ✅ Можно ли отозвать согласие через Telegram бота (команда /cancel_consent)?
- ✅ Достаточно ли разместить политику конфиденциальности на сайте и считать, что пользователь ознакомился?

**Почему это критично:** Без правильного механизма получения согласия обработка ПД незаконна. Аудитор ОАЦ проверит наличие механизма получения согласия.

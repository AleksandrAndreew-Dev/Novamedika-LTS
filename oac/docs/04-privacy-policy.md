## 2. ПОРЯДОК И УСЛОВИЯ ОБРАБОТКИ ПЕРСОНАЛЬНЫХ ДАННЫХ

### 2.1. Перечень действий с персональными данными

Оператор осуществляет следующие действия с персональными данными:
- Сбор — получение ПД от субъектов при регистрации, оформлении заказа, загрузке данных
- Хранение — сохранение ПД в базе данных PostgreSQL и Redis
- Использование — обработка для предоставления услуг Сервиса
- Уточнение (обновление, изменение) — по запросу субъекта или при обновлении данных
- Удаление — удаление ПД по запросу субъекта или по истечении срока хранения
- Блокирование — временное прекращение обработки при запросе субъекта

**Источник получения персональных данных:** непосредственно от субъектов персональных данных при использовании Сервиса.

### 2.2. Цели обработки персональных данных

#### Цель 1: Регистрация и обслуживание фармацевтов

| Параметр | Значение |
|----------|----------|
| **Категория субъектов** | Кандидаты на трудоустройство (фармацевты) |
| **Перечень ПД** | Telegram ID, имя, фамилия, отчество, номер телефона, статус регистрации, данные о квалификации, время онлайн/офлайн |

| **Срок хранения** | 1 год после прекращения сотрудничества (дата удаления аккаунта или прекращения активности) |
| **Описание процесса** | Фармацевт проходит регистрацию в Telegram-боте с использованием секретного слова. После регистрации может отвечать на вопросы пользователей, менять статус онлайн/офлайн, экспортировать историю консультаций. |

#### Цель 2: Проведение онлайн-консультаций (Q&A система)

| Параметр | Значение |
|----------|----------|

| **Перечень ПД** | Telegram ID, имя, фамилия, номер телефона (при предоставлении), текст вопроса, текст ответа, история диалога |
| **Правовое основание** | Статья 6 Закона №99-З — согласие субъекта персональных данных |
| **Срок хранения** | 1 год после последнего обращения (дата последнего вопроса или ответа) |
| **Описание процесса** | Пользователь задаёт вопрос через Telegram-бота. Вопрос передаётся свободному фармацевту. Фармацевт отвечает. Диалог сохраняется в базе данных. |

#### Цель 3: Оформление заказов на лекарства

| Параметр | Значение |
|----------|----------|
| **Категория субъектов** | Покупатели (заказчики) |

| **Описание процесса** | Пользователь выбирает лекарство на сайте, оформляет бронь в конкретной аптеке. Заказ передаётся в аптеку. Аптека подтверждает наличие. |

#### Цель 4: Синхронизация данных с аптеками-партнёрами

| Параметр | Значение |
|----------|----------|
| **Категория субъектов** | Представители аптек-партнёров (контрагенты) |
| **Перечень ПД** | Наименование аптеки, номер аптеки, адрес, режим работы, данные из CSV-файлов (наименование лекарств, форма, производитель, цена, наличие, срок годности) |
| **Правовое основание** | Статья 6 Закона №99-З — исполнение договора |
| **Срок хранения** | До следующей синхронизации (дата последней синхронизации + 30 дней) |
| **Описание процесса** | Аптека-партнёр загружает CSV-файл через API или данные синхронизируются автоматически через Tabletka.by API. Система обновляет информацию о наличии и ценах. |

| **Правовое основание** | Статья 6 Закона №99-З — законный интерес Оператора; требования законодательства об информации, информатизации и защите информации |
| **Срок хранения** | 1 год с даты создания записи |
| **Описание процесса** | Сервис автоматически регистрирует события аутентификации, доступа, ошибки, подозрительную активность. Логи хранятся в журналах Docker-контейнеров. |

### 2.4. Применяемые меры защиты персональных данных

Оператор принимает следующие правовые, организационные и технические меры для обеспечения защиты персональных данных в соответствии со статьей 16 Закона № 99-З и требованиями Приказа ОАЦ № 66 (в редакции Приказа ОАЦ № 195):

**Правовые меры:**
- Настоящая Политика обработки персональных данных
- Локальные правовые акты по вопросам защиты информации (политика ИБ, регламенты)
- Договоры с уполномоченными лицами (при наличии)
- Акт отнесения информационной системы к классу 3-ин

**Организационные меры:**
- Разграничение доступа к персональным данным по принципу минимальных привилегий (роли: user, pharmacist, admin)
- Обучение работников правилам работы с персональными данными
- Контроль за соблюдением требований по защите ПД
- Определение сроков хранения ПД для каждой цели обработки (см. раздел 2.2)
- Назначение ответственного за обеспечение защиты информации (или выполнение работ самостоятельно для ИП)
- Регламентация порядка использования съемных носителей и мобильных технических средств
- Определение перечня разрешенного программного обеспечения

**Технические меры:**

*Идентификация и аутентификация:*
- JWT-аутентификация (access token 30 мин, refresh token 7 дней)
- Защита обратной связи при вводе пароля (masked input fields)
- Блокировка доступа после истечения времени бездействия (session timeout)
- Полномочное управление учетными записями (создание, активация, блокировка, удаление)
- Контроль соблюдения правил генерации и смены паролей (минимум 12 символов, сложность)

*Шифрование и криптографическая защита:*
- Шифрование передаваемых данных (HTTPS/TLS 1.2+ через Let's Encrypt)
- Шифрование чувствительных персональных данных в БД (AES-256 через pgcrypto):
  - Номера телефонов пользователей и фармацевтов
  - Telegram ID пользователей и фармацевтов
- Шифрование API-токенов аптек (Fernet/AES-128-CBC)
- Управление ключами шифрования согласно Регламенту управления ключами
- **Примечание:** Согласно пункту 19¹ Приказа ОАЦ № 195, если ПД получены без криптозащиты, их возврат субъекту может осуществляться также без криптозащиты

*Разграничение доступа:*
- Ролевая модель доступа (RBAC): user, pharmacist, admin
- Middleware для проверки ролей на уровне эндпоинтов
- Работа контейнеров от непривилегированных пользователей (non-root)
- HTTP Basic Auth для административных эндпоинтов

*Защита инфраструктуры:*
- Docker resource limits для изоляции контейнеров (CPU, memory)
- Security headers (CSP, HSTS, X-Frame-Options, XSS-Filter, X-Content-Type-Options)
- Защита БД паролем (PostgreSQL AUTH, уникальные credentials)
- Защита Redis паролем (Redis AUTH)
- Межсетевое экранирование (Traefik как reverse proxy)
- Синхронизация времени (NTP) для корректной временной метки событий

*Мониторинг и аудит:*
- Регистрация событий информационной безопасности:
  - События аутентификации (успешные/неуспешные входы)
  - Нарушения прав доступа
  - Подозрительная активность
  - Ошибки приложения
- Централизованный сбор и хранение логов (Docker logging driver)
- Срок хранения логов: **не менее 1 года** (требование Приказа ОАЦ № 66, приложение 3)
- Мониторинг событий ИБ уполномоченными лицами
- Автоматизированное оповещение о критических событиях

*Резервное копирование:*
- Регулярное резервное копирование базы данных PostgreSQL
- Защита резервных копий от несанкционированного доступа
- Проверка целостности резервных копий
- Срок хранения резервных копий: согласно Регламенту резервного копирования

*Антивирусная защита и обновление:*
- Регулярное обновление системного ПО и зависимостей (security patches)
- Антивирусная защита серверной инфраструктуры
- Обновление сигнатур антивируса
- Сканирование уязвимостей (periodic vulnerability scanning)
- Контроль состава объектов информационной системы

*Защита от сетевых атак:*
- Rate limiting для предотвращения DDoS
- CORS policy (ограничение источников запросов)
- Input validation и sanitization (защита от SQL injection, XSS)
- Обнаружение и предотвращение вторжений (IDS/IPS на уровне reverse proxy)

- Написать запрос на email: **[privacy@novamedika.com]**
- Использовать Telegram-бот: команда `/my_data` (в разработке)
- Заполнить форму на сайте: spravka.novamedika.com/privacy (в разработке)

Оператор рассматривает заявление в течение **15 рабочих дней** со дня получения.

### 5.2. Право на изменение персональных данных

Субъект ПД имеет право требовать уточнения (изменения) своих персональных данных, если они являются неполными, устаревшими или неточными.

### 5.3. Право на удаление персональных данных

Субъект ПД имеет право требовать удаления своих персональных данных, если:

- Субъект отозвал согласие
- Обработка осуществляется с нарушением законодательства

**Как реализовать:**
- Написать запрос на email: **[privacy@novamedika.com]**
- Telegram-бот: команда `/delete_account` (в разработке)

Оператор удаляет ПД в течение **30 календарных дней** со дня получения заявления.

**Ограничения:** Оператор вправе сохранить ПД, если обработка необходима для исполнения законодательства (например, хранение логов в течение 1 года).

После отзыва согласия Оператор прекращает обработку ПД и удаляет их в соответствии со сроками, указанными в разделе 2.

## 6. ВНЕШНИЕ СЕРВИСЫ И ССЫЛКИ

6.1. Сервис NovaMedika2 содержит ссылки на внешние ресурсы, которые открываются по инициативе пользователя.

6.2. При переходе по внешним ссылкам:

- Переход осуществляется исключительно по явной инициативе пользователя (клик по кнопке/ссылке)
- Внешние ссылки открываются в новой вкладке браузера с атрибутами безопасности `rel="noopener noreferrer"`
- Передаются только публичные данные (например, адрес аптеки для отображения на карте)
- **Персональные данные пользователей НЕ передаются** на внешние ресурсы

6.3. Используемые внешние сервисы:

| Внешний сервис | Назначение | Передаваемые данные |
|---------------|------------|---------------------|
| Google Maps / Yandex Maps / OpenStreetMap | Отображение расположения аптек на карте | Адрес аптеки (публичные данные) |

6.4. Оператор не контролирует и не несёт ответственность за:

- Содержимое внешних сайтов
- Политику конфиденциальности внешних сервисов
- Практики сбора данных внешними сервисами

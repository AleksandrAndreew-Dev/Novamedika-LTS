## 1. ОБЩИЕ ПОЛОЖЕНИЯ

1.1. Настоящая Политика информационной безопасности (далее — Политика) определяет цели, принципы и обязательства Организации по обеспечению защиты информации в информационной системе **NovaMedika2** (далее — ИС, Система).

1.2. ИС NovaMedika2 — справочный сервис поиска лекарственных средств, оформления заказов и проведения онлайн-консультаций между пользователями и фармацевтами.

1.3. Класс типовых информационных систем: **3-ин** (персональные данные + открытые каналы передачи данных).

1.4. Настоящая Политика обязательна для исполнения всеми работниками Организации, имеющими доступ к активам ИС NovaMedika2.

1.5. Политика доводится до сведения работников в части, их касающейся, и доступна заинтересованным субъектам информационных отношений для ознакомления.

## 2. ЦЕЛИ И ПРИНЦИПЫ ЗАЩИТЫ ИНФОРМАЦИИ

### 2.1. Цели

- Обеспечение конфиденциальности персональных данных пользователей и фармацевтов
- Обеспечение целостности и доступности данных ИС
- Предотвращение несанкционированного доступа к информации
- Выполнение требований законодательства Республики Беларусь в области защиты информации
- Соответствие требованиям приказа ОАЦ №66 от 20.02.2020

### 2.2. Принципы

- **Законность** — обработка информации в соответствии с законодательством
- **Минимальные привилегии** — доступ к информации предоставляется в минимально необходимом объёме
- **Непрерывность совершенствования** — постоянное улучшение системы защиты информации
- **Комплексность** — применение совокупности правовых, организационных и технических мер
- **Прозрачность** — информирование субъектов ПД о порядке обработки их данных

## 4. СИСТЕМА ЗАЩИТЫ ИНФОРМАЦИИ

### 4.1. Состав системы защиты информации

Система защиты информации ИС NovaMedika2 включает совокупность средств и мер:

**Средства криптографической защиты:**
- TLS-шифрование (Let's Encrypt, ACME) для защиты передаваемых данных
- JWT-аутентификация (access token 30 мин, refresh token 7 дней)
- Fernet-шифрование для защиты чувствительных данных в БД (токены API аптек)

**Средства защиты от НСД:**
- Аутентификация: JWT Bearer, HTTP Basic Auth, API Keys, Redis AUTH, PostgreSQL AUTH
- Разграничение доступа: роли user / pharmacist / admin
- Docker non-root — все контейнеры от непривилегированных пользователей
- Docker resource limits — изоляция ресурсов контейнеров

**Средства контроля целостности:**
- HTTPS/TLS для контроля целостности передаваемых данных
- Health checks для контроля работоспособности контейнеров
- Мониторинг событий ИБ

**Средства межсетевого экранирования:**
- Traefik v3.6 — обратный прокси с маршрутизацией
- Docker bridge network — сегментация внутренней сети
- Security headers: CSP, HSTS, X-Frame-Options, XSS-Filter, Referrer-Policy

### 4.2. Организационные меры

- Назначение ответственного за обеспечение защиты информации
- Разработка локальных правовых актов по вопросам применения системы защиты
- Проведение инструктажей работников не реже одного раза в год
- Анализ эффективности применения системы защиты не реже одного раза в год
- Резервное копирование данных и конфигураций

## 5. ПРАВА И ОБЯЗАННОСТИ ПОЛЬЗОВАТЕЛЕЙ ИНФОРМАЦИОННОЙ СИСТЕМЫ

### 5.1. Пользователи ИС (работники Организации) обязаны:

- Соблюдать требования настоящей Политики
- Не передавать свои учётные данные (пароли, токены) третьим лицам
- Немедленно сообщать ответственному за защиту информации о выявленных нарушениях
- Использовать только разрешённое программное обеспечение
- Соблюдать порядок работы с персональными данными в соответствии с Политикой обработки ПД

### 5.2. Пользователи ИС (работники Организации) вправе:

- Получить информацию о мерах защиты информации в части, их касающейся
- Обратиться к ответственному за защиту информации за разъяснениями
- Сообщить о выявленных уязвимостях и нарушениях

### 5.3. Запрещается:

- Использовать учётные записи с административными привилегиями для повседневной работы
- Устанавливать неразрешённое программное обеспечение на сервер
- Передавать доступ к системе защиты информации третьим лицам
- Отключать средства защиты информации без согласования с ответственным

## 6. ПОРЯДОК ПРИМЕНЕНИЯ СИСТЕМЫ ЗАЩИТЫ ИНФОРМАЦИИ

### 6.1. Выявление угроз и реагирование на события ИБ

| Тип события | Действия | Ответственный | Срок |
|------------|----------|--------------|------|
| Несанкционированный доступ | Блокировка учётной записи, расследование | Администратор | Немедленно |
| Сбой функционирования ИС | Восстановление из резервной копии | Администратор | В течение 4 часов |
| Утечка персональных данных | Информирование ОАЦ, НЦЗПД | Ответственный за защиту | В течение 24 часов |
| Компрометация криптографических ключей | Прекращение использования, перевыпуск | Администратор | Немедленно |
| Нарушение политики ИБ | Расследование, устранение нарушений | Ответственный за защиту | В течение 5 рабочих дней |

### 6.2. Порядок управления учётными записями

- Создание учётной записи — по приказу руководителя или через процедуру регистрации
- Блокировка — при увольнении, нарушении правил, неактивности более 90 дней
- Удаление — по истечении 1 года после блокировки
- Контроль паролей: минимальная длина 12 символов, регулярная смена

### 6.3. Порядок обновления программного обеспечения

- Обновление Docker-образов — через CI/CD pipeline (GitHub Actions)
- Обновление зависимостей — еженедельная проверка (Dependabot)
- Тестирование обновлений перед внедрением в production
- Контроль своевременности — ответственный администратор

### 6.4. Порядок резервного копирования

| Объект | Метод | Периодичность | Хранение |
|--------|-------|--------------|----------|
| PostgreSQL БД | pg_dump | Ежедневно | 30 дней |
| Конфигурации (docker-compose, nginx, traefik) | Копирование | При изменении | Бессрочно |
| TLS-сертификаты | Автоматически (Let's Encrypt) | При обновлении | — |
| Логи контейнеров | Docker json-file | Непрерывно | 365 дней |

### 6.5. Синхронизация системного времени

Системное время всех контейнеров синхронизируется от единого источника (NTP на хост-сервере).

- **Ответственный:** ответственный за защиту информации
- **Результат:** документ произвольной формы, утверждённый руководителем

### 7.3. Инструктажи работников

- **Периодичность:** не реже одного раза в год
- **Тематика:** правила работы с ПД, реагирование на инциденты, актуальные угрозы
- **Ответственный:** ответственный за защиту информации

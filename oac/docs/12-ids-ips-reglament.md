## 1. ОБЩИЕ ПОЛОЖЕНИЯ

## 2. СРЕДСТВА ОБНАРУЖЕНИЯ ВТОРЖЕНИЙ

### 2.1. Сетевой уровень

| Средство | Назначение | Конфигурация |
|----------|-----------|-------------|
| Fail2ban | Блокировка IP при подозрительной активности | jail.local |
| Docker networks | Сегментация, изоляция контейнеров | traefik-public (bridge) |
| Traefik rate limiting | Защита от DDoS | Middleware (настраивается) |

### 2.2. Уровень приложения

| Средство | Назначение | Конфигурация |
|----------|-----------|-------------|
| Traefik security headers | Защита от XSS, clickjacking | CSP, HSTS, X-Frame |
| JWT валидация | Защита от подделки токенов | HMAC-SHA256, 30 мин TTL |
| Rate limiting (slowapi) | Защита от брутфорса API | 30/min на /search-fts |
| Мониторинг ошибок | Обнаружение аномалий | Loki + Grafana |

### 2.3. Уровень базы данных

| Средство | Назначение |
|----------|-----------|
| PostgreSQL AUTH | Защита от несанкционированного доступа |
| Шифрование pgcrypto | Защита данных при хранении |
| Decryption audit log | Регистрация расшифровки ПД |

## 3. ПРАВИЛА ОБНАРУЖЕНИЯ

### 3.1. Критерии подозрительной активности

| Критерий | Порог | Действие |
|----------|-------|----------|
| Неудачные попытки аутентификации | >5 за 10 минут | Fail2ban: бан IP 1 час |
| Запросы к несуществующим эндпоинтам | >20 за 1 минуту | Fail2ban: бан IP 1 час |
| Аномальный объём запросов | >100 за 1 минуту | Rate limiting + логирование |
| SQL-инъекция (паттерны) | Любое совпадение | Блокировка + CRITICAL алерт |
| XSS попытка | Любое совпадение | Блокировка + HIGH алерт |
| Доступ к /upload/ без Basic Auth | Любое совпадение | Блокировка + WARN алерт |

### 3.2. Правила Fail2ban

| Jail | Фильтр | Порт | MaxRetry | Ban Time |
|------|--------|------|----------|----------|
| sshd | sshd | 22 | 3 | 2 часа |
| nginx-http-auth | nginx-http-auth | 80,443 | 5 | 1 час |
| nginx-badbots | nginx-badbots | 80,443 | 3 | 24 часа |
| nginx-404 | nginx-noscript | 80,443 | 20 | 1 час |
| nginx-limit-req | nginx-limit-req | 80,443 | 10 | 1 час |

## 4. РЕАГИРОВАНИЕ НА ВТОРЖЕНИЯ

### 4.1. Уровни инцидентов

| Уровень | Описание | Время реакции | Эскалация |
|---------|----------|--------------|-----------|
| INFO | Подозрительная активность | 24 часа | Администратор |
| WARN | Попытка вторжения | 4 часа | Администратор |
| HIGH | Успешное вторжение | 1 час | Ответственный за защиту ИБ |
| CRITICAL | Компрометация данных | 15 минут | Руководитель + ОАЦ |

### 4.2. Порядок реагирования

**WARN (попытка вторжения):**
1. Заблокировать IP (Fail2ban)
2. Зафиксировать в журнале
3. Проверить, не было ли успешного доступа

**HIGH (успешное вторжение):**
1. Изолировать затронутый контейнер
2. Сохранить логи для расследования
3. Уведомить ответственного за защиту ИБ
4. Начать расследование

**CRITICAL (компрометация):**
1. Немедленно изолировать все затронутые компоненты
2. Сменить все ключи и пароли
3. Уведомить руководителя в течение 15 минут
4. Уведомить ОАЦ в течение 24 часов (если затронуты ПД)
5. Провести расследование в течение 5 рабочих дней

## 5. МОНИТОРИНГ

### 5.1. Ежедневный мониторинг

- Просмотр Grafana дашборда «Security Monitoring»
- Проверка заблокированных IP (Fail2ban status)
- Проверка алертов в Loki

### 5.2. Еженедельный анализ

- Анализ попыток вторжений
- Обновление правил Fail2ban (при необходимости)
- Проверка актуальности Traefik security headers

### 5.3. Команды мониторинга

```bash
# Fail2ban статус
docker exec fail2ban-prod fail2ban-client status

# Fail2ban конкретный jail
docker exec fail2ban-prod fail2ban-client status sshd

# Заблокированные IP
docker exec fail2ban-prod fail2ban-client status nginx-http-auth

# Loki: поиск ошибок
# Grafana: https://grafana.novamedika.com
```

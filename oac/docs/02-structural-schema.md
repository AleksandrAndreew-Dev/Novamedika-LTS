## 11. Зоны ответственности (хостер vs наша команда)

Инфраструктура ИС разделена на две зоны обслуживания. Детальное распределение задач см. в [HOSTER-DIVISION-OF-WORK.md](../planning/HOSTER-DIVISION-OF-WORK.md).

| Зона | Компоненты | Ответственный |
|------|------------|----------------|
| **Инфраструктурный уровень** | Сервер, МЭ, сеть, NTP, виртуализация, физическая защита | 🔵 ХОСТЕР |
| **Инфраструктурный мониторинг** | SIEM, ELK Stack, инфраструктурные метрики | 🔵 ХОСТЕР |
| **Backup (хранение)** | S3 storage, retention 365 дней, шифрование backup | 🔵 ХОСТЕР |
| **Active Directory** | Аутентификация, групповые политики | 🔵 ХОСТЕР |
| **Антивирус / IDS/IPS** | Установка, настройка, мониторинг | 🔵 ХОСТЕР |
| **Интеграция СЗИ** | Подключение СЗИ в инфраструктуру | 🔵 ХОСТЕР |
| **Прикладной уровень** | Backend, Frontend, PostgreSQL, Redis, Celery | 🟢 МЫ |
| **Прикладная безопасность** | pgcrypto, Fernet, JWT, audit middleware, 2FA | 🟢 МЫ |
| **Отправка логов в SIEM** | Конфигурация Fluent Bit / Promtail | 🟢 МЫ |
| **Прикладной мониторинг** | Prometheus /metrics, Grafana dashboards | 🟢 МЫ |
| **Документация и аттестация** | Политики, схемы, ТЗ, акты, ОАЦ | 🟢 МЫ |
| **Логирование (контент)** | Содержание логов, формат JSON, отправка | 🟡 СОВМЕСТНО |
| **Структурная/логическая схема** | Данные по нашим сервисам + инфраструктура хостера | 🟡 СОВМЕСТНО |
| **Backup (тестирование)** | Тестирование восстановления нашего приложения | 🟡 СОВМЕСТНО |
| **NTP-подключение** | Настройка NTP-клиента на наших контейнерах | 🟡 СОВМЕСТНО |

## 🔐 Что шифруется

### Зашифрованные поля (прямые идентификаторы):

| Поле | Таблица | Метод | Причина |
|------|---------|-------|---------|
| `telegram_id` | `qa_users` | `encrypt_bigint()` | Уникальный ID пользователя |
| `phone` | `qa_users` | `encrypt_value()` | Номер телефона |
| `customer_phone` | `booking_orders` | `encrypt_value()` | Телефон заказчика |
| `telegram_id` | `booking_orders` | `encrypt_bigint()` | ID пользователя в заказе |

### НЕ шифруются:

- Имена (`first_name`, `last_name`, `customer_name`)
- Данные аптек
- Информация о лекарствах

**Обоснование:** Принцип пропорциональности - шифруются только прямые идентификаторы

## 🛠️ Технические детали

### Алгоритм:
- **Метод:** Fernet (symmetric encryption)
- **Библиотека:** `cryptography.fernet`
- **Алгоритм:** AES-128-CBC с HMAC-SHA256
- **Размер ключа:** 32 байта (256 бит)
- **Кодировка:** Base64

### PostgreSQL:
- **Расширение:** pgcrypto v1.3+
- **Функции:** `pgp_sym_encrypt()`, `pgp_sym_decrypt()`

### Архитектура:
- **Подход:** Гибридный (открытое + зашифрованное поле)
- **Причина:** Быстрый поиск + безопасное хранение
- **План:** 3 фазы миграции

- 📅 Каждые 6-12 месяцев (плановая)

## 🔄 Процесс отката

Если что-то пошло не так:

```bash
# 1. Восстановить БД из бэкапа
cat backup_YYYYMMDD_HHMMSS.sql | \
  docker exec -i postgres-prod psql -U $POSTGRES_USER -d $POSTGRES_DB

# 2. Откатить миграцию
docker exec -it backend-prod alembic downgrade -1

# 3. Перезапустить сервисы
docker compose -f docker-compose.traefik.prod.yml up -d
```

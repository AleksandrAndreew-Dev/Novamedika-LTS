## 🎯 Ключевые моменты для запоминания

### ✅ ДЕЛАТЬ:
- ✓ Всегда делать бэкап перед миграцией
- ✓ Сохранять ключ шифрования в безопасном месте
- ✓ Использовать разные ключи для разных сред
- ✓ Добавлять .env в .gitignore
- ✓ Тестировать шифрование после настройки
- ✓ Мониторить логи первые дни после внедрения

### ❌ НЕ ДЕЛАТЬ:
- ✗ Никогда не коммитьте .env в Git
- ✗ Не используйте один ключ для dev/prod
- ✗ Не удаляйте старые поля сразу (гибридный подход)
- ✗ Не игнорируйте ошибки в логах
- ✗ Не забывайте про ротацию ключей

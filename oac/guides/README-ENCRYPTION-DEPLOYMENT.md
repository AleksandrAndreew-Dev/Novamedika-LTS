## 🔐 Что шифруется?

### ✅ Зашифрованные поля (прямые идентификаторы):

| Поле | Таблица | Тип данных | Метод шифрования |
|------|---------|------------|------------------|
| `telegram_id` | `qa_users` | BigInt | `encrypt_bigint()` |
| `phone` | `qa_users` | String | `encrypt_value()` |
| `customer_phone` | `booking_orders` | String | `encrypt_value()` |
| `telegram_id` | `booking_orders` | BigInt | `encrypt_bigint()` |

### ❌ НЕ шифруются (не являются прямыми идентификаторами):

- Имена пользователей (`first_name`, `last_name`, `customer_name`)
- Данные аптек (названия, адреса, телефоны)
- Информация о лекарствах (названия, цены, наличие)

**Обоснование:** Согласно принципу пропорциональности, шифруются только те данные, которые позволяют однозначно идентифицировать физическое лицо.

## 🛠️ Технические детали

### Алгоритм шифрования:
- **Метод:** Fernet (symmetric encryption)
- **Библиотека:** `cryptography.fernet`
- **Алгоритм:** AES-128-CBC с HMAC-SHA256
- **Размер ключа:** 32 байта (256 бит)
- **Кодировка:** Base64

### Расширение PostgreSQL:
- **Название:** pgcrypto
- **Версия:** 1.3+
- **Функции:** `pgp_sym_encrypt()`, `pgp_sym_decrypt()`

### Архитектура:
- **Гибридный подход:** Храним оба поля (открытое + зашифрованное)
- **Причина:** Быстрый поиск по открытому полю + безопасное хранение
- **План миграции:** 3 фазы (текущая → переходная → полная)

## 📊 Ожидаемые метрики

### Производительность:
- **Overhead шифрования:** +5-10ms на операцию
- **Увеличение размера данных:** ~30% (base64 encoding)
- **Использование CPU:** +2-5%
- **Использование памяти:** +10-20MB

### Безопасность:
- **Защищенные поля:** 4 (в 2 таблицах)
- **Соответствие:** ОАЦ класс 3-ин ✓
- **Стандарт:** AES-128-CBC с HMAC-SHA256

## ⚠️ Важные предупреждения

### 1. Безопасность ключа

```bash
# НИКОГДА не коммитьте .env в Git!
echo ".env" >> .gitignore
git add .gitignore
git commit -m "Add .env to gitignore"

# Проверяйте перед каждым коммитом
git status
git diff --cached
```

### 2. Хранение ключа

- ✅ Сохраните ключ в менеджере паролей (1Password, Bitwarden, KeePass)
- ✅ Используйте HashiCorp Vault или AWS Secrets Manager для production
- ✅ Делайте бэкапы ключей в безопасном месте
- ❌ Не храните ключ в коде
- ❌ Не передавайте ключ через незащищенные каналы
- ❌ Не используйте один ключ для разных сред

### 3. Ротация ключей

**Рекомендуемый график:**
- Каждые 6-12 месяцев плановая ротация
- Немедленно при подозрении на компрометацию
- После увольнения сотрудника с доступом

**Процесс ротации:**
1. Сгенерировать новый ключ
2. Перешифровать все данные новым ключом
3. Обновить конфигурацию всех сервисов
4. Удалить старый ключ
5. Протестировать работоспособность

## 🔄 Процесс отката

Если что-то пошло не так:

```bash
# 1. Остановить сервисы
docker compose -f docker-compose.traefik.prod.yml down

# 2. Восстановить БД из бэкапа
cat backup_YYYYMMDD_HHMMSS.sql | \
  docker exec -i postgres-prod psql \
  -U $POSTGRES_USER -d $POSTGRES_DB

# 3. Откатить миграцию
docker exec -it backend-prod alembic downgrade -1

# 4. Удалить ENCRYPTION_KEY из .env
sed -i '/ENCRYPTION_KEY/d' .env

# 5. Перезапустить сервисы
docker compose -f docker-compose.traefik.prod.yml up -d

# 6. Проверить работоспособность
curl -f https://api.novamedika.com/health
docker logs backend-prod --tail 50
```

### 🎯 Принципы, которым следовали

1. **Оригинал ≠ Проектный документ** — `origin-docs/` для текстов законов, `oac/docs/` для compliance
2. **Один тип = одна папка** — гайды → `guides/`, аудиты → `audits/`, планы → `planning/`
3. **Соответствие = нумерация** — `oac/docs/` строго 01-15
4. **OAC ≠ Проект** — ОАЦ-compliance → `oac/`, прочая документация → `docs/` в корне
5. **Telegram-специфика = подпапка OAC** — `oac/telegram/` вместо изолированной папки в корне

### ⚠️ Возможные риски

- **Сломанные относительные ссылки** в файлах, которые ссылались на старые пути
- **Параллельная работа** — если кто-то работал со старой структурой
- **Git история** — перемещение теряет git blame (для важных файлов можно сделать `git mv`)

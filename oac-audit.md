## 1. Аудит безопасности (Раздел 2.1)

### 1.1 Регистрация событий ИБ (логи)
**Требование:** Регистрация событий информационной безопасности  
**Статус:** ✅ **ЧАСТИЧНО РЕАЛИЗОВАНО**

**Реализовано:**
- ✅ Логирование в Docker контейнерах (backend, frontend, traefik, postgres, redis, celery)
- ✅ Логи доступны через `docker logs`
- ✅ Healthcheck endpoints для мониторинга состояния сервисов
- ✅ Logging middleware в FastAPI приложениях

**Не реализовано:**
- ❌ Централизованный сбор логов (ELK Stack, Graylog или аналог)
- ❌ Хранение логов ≥ 1 года (сейчас логи теряются при перезапуске контейнеров)
- ❌ Ротация и архивация логов
- ❌ Поиск и анализ логов
- ❌ Мониторинг событий ИБ в реальном времени

**Рекомендация:** Внедрить стек ELK (Elasticsearch + Logstash + Kibana) или Graylog для централизованного сбора логов с retention policy 1 год.

---

### 1.2 Хранение логов ≥ 1 года
**Требование:** Хранение логов не менее 1 года  
**Статус:** ❌ **НЕ РЕАЛИЗОВАНО**

**Текущее состояние:**
- Логи хранятся только в stdout/stderr контейнеров
- При перезапуске контейнеров логи теряются
- Нет механизма ротации и архивации
- Нет резервного копирования логов

**Требуется реализовать:**
1. Настроить volume для persistent storage логов
2. Внедрить систему ротации (logrotate)
3. Настроить архивацию старых логов
4. Реализовать backup логов на внешнее хранилище
5. Установить retention policy: 1 год минимум

**Приоритет:** 🔴 **КРИТИЧНЫЙ**

---

### 1.3 Централизованный сбор логов
**Требование:** Централизованный сбор логов ≥ 1 года  
**Статус:** ❌ **НЕ РЕАЛИЗОВАНО**

**Рекомендуемое решение:**
```yaml
# Добавить в docker-compose:
services:
  elasticsearch:
    image: elasticsearch:8.x
    volumes:
      - es-data:/usr/share/elasticsearch/data
    environment:
      - discovery.type=single-node
      
  logstash:
    image: logstash:8.x
    volumes:
      - ./logstash/pipeline:/usr/share/logstash/pipeline
    depends_on:
      - elasticsearch
      
  kibana:
    image: kibana:8.x
    ports:
      - "5601:5601"
    depends_on:
      - elasticsearch
```

**Приоритет:** 🔴 **КРИТИЧНЫЙ**

**Реализация:**
```python
# Пример для Telegram бота

# В middleware проверять last_activity_time
if (current_time - user.last_activity_time) > FSM_TIMEOUT:
    await state.clear()
    await message.answer("Сеанс завершён из-за неактивности")
```

### 5.2 Шифрование при хранении
**Статус:** ❌ **НЕ РЕАЛИЗОВАНО**

**Требуется:**
1. Transparent Data Encryption (TDE) для PostgreSQL
2. Или шифрование на уровне filesystem (LUKS)
3. Или шифрование чувствительных полей в БД (pgcrypto)

**Рекомендация для PostgreSQL:**
```sql
-- Установка расширения pgcrypto
CREATE EXTENSION IF NOT EXISTS pgcrypto;

-- Шифрование чувствительных данных
ALTER TABLE users 
ADD COLUMN phone_encrypted bytea;

UPDATE users 
SET phone_encrypted = pgp_sym_encrypt(phone, 'encryption_key');
```

**Приоритет:** 🔴 **КРИТИЧНЫЙ**

---

### 5.3 Электронная цифровая подпись (ЭЦП)

# WebApp Dashboard для фармацевтов - Консультации

## 📖 Обзор

WebApp Dashboard - это профессиональный веб-интерфейс для фармацевтов, позволяющий эффективно **консультировать пользователей** через Telegram Bot.

### Ключевые возможности

✅ **Система консультаций**
- Чат-интерфейс для общения с пользователями
- История диалогов с временными метками
- Возможность прикреплять фото рецептов
- Быстрые шаблоны ответов
- Real-time уведомления о новых вопросах

✅ **Управление вопросами**
- Фильтрация по статусам (новые, в работе, завершенные)
- Поиск по истории консультаций
- Назначение вопросов себе
- Завершение консультаций

✅ **Дашборд и статистика**
- Количество новых вопросов
- Вопросы в работе
- Завершенные консультации за сегодня
- Среднее время ответа

✅ **Real-time обновления**
- WebSocket подключение для мгновенных уведомлений
- Автоматическое переподключение при разрыве связи
- Синхронизация статуса онлайн/офлайн
- Браузерные уведомления

## 🚀 Быстрый старт

### Предварительные требования

- Node.js 18+ 
- npm или yarn
- Backend API должен быть запущен

### Установка

1. **Установить зависимости frontend**
```bash
cd frontend
npm install
```

2. **Настроить переменные окружения**

Создайте файл `.env` в директории `frontend/`:

```env
# API URL
VITE_API_URL=http://localhost:8000

# WebSocket URL
VITE_WS_URL=ws://localhost:8000/ws/pharmacist

# Mode (development or production)
NODE_ENV=development
```

3. **Запустить development сервер**
```bash
npm run dev
```

Приложение будет доступно по адресу: `http://localhost:5173`

## 📱 Использование

### Вход в систему

1. Откройте приложение в браузере
2. Введите ваш **Telegram ID** и **пароль**
3. Нажмите "Войти"

> **Примечание:** Учетные данные выдаются администратором системы

### Работа с консультациями

#### Просмотр вопросов
1. Перейдите в раздел **"Консультации"**
2. Используйте фильтры для поиска нужных вопросов:
   - **Новые** - требуют вашего ответа
   - **В работе** - вы уже отвечаете
   - **Завершенные** - закрытые консультации

#### Ответ на вопрос
1. Выберите вопрос из списка
2. Прочитайте сообщение пользователя
3. При необходимости просмотрите прикрепленное фото рецепта
4. Напишите ответ в чате
5. Используйте **быстрые шаблоны** для частых ответов
6. Нажмите "Отправить"

#### Завершение консультации
1. После предоставления всей информации нажмите "Завершить консультацию"
2. Вопрос перейдет в статус "completed"
3. Пользователь получит уведомление о завершении

### Статус онлайн/офлайн

- Переключайте статус в боковом меню
- В режиме **онлайн** вы получаете уведомления о новых вопросах
- В режиме **офлайн** вопросы не назначаются вам автоматически

### API Endpoints

#### Консультации
```
GET    /api/pharmacist/questions              # Список вопросов
GET    /api/pharmacist/questions/{id}         # Детали вопроса
POST   /api/pharmacist/questions/{id}/answer  # Отправить ответ
PUT    /api/pharmacist/questions/{id}/complete # Завершить
POST   /api/pharmacist/questions/{id}/assign  # Назначить себе
GET    /api/pharmacist/questions/unread-count # Счетчик непрочитанных
GET    /api/pharmacist/quick-replies          # Шаблоны ответов
GET    /api/pharmacist/consultations/stats    # Статистика
```

#### WebSocket Events
```
Server → Client:
  - new_question        # Новый вопрос от пользователя
  - question_answered   # Вопрос получил ответ
  - question_completed  # Консультация завершена
```

## 🔐 Безопасность

### Аутентификация

- JWT токены с автоматическим обновлением
- Access token: 30 минут
- Refresh token: 7 дней
- Автоматический logout при истечении токенов

### Защита данных

- Все персональные данные шифруются в БД (pgcrypto)
- HTTPS соединение (TLS 1.3)
- CORS policy ограничивает доступ к API
- Rate limiting предотвращает brute-force атаки

## 🐛 Решение проблем

### Проблема: Не могу войти в систему

**Решение:**
1. Проверьте правильность Telegram ID и пароля
2. Убедитесь, что backend API запущен
3. Проверьте консоль браузера на ошибки
4. Обратитесь к администратору для сброса пароля

### Проблема: WebSocket не подключается

**Решение:**
1. Проверьте URL WebSocket в `.env` файле
2. Убедитесь, что backend поддерживает WebSocket
3. Проверьте firewall и сетевые настройки
4. Обновите страницу (F5)

### Проблема: Не приходят уведомления о новых вопросах

**Решение:**
1. Убедитесь, что вы в режиме "онлайн"
2. Проверьте подключение к интернету
3. Разрешите уведомления в браузере
4. Проверьте консоль на ошибки WebSocket

### Проблема: Медленная загрузка страницы

**Решение:**
1. Очистите кэш браузера
2. Проверьте скорость интернета
3. Закройте лишние вкладки
4. Попробуйте другой браузер
